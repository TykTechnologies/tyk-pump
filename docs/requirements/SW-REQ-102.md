# SW-REQ-102: Mongo aggregate Lists projection is preserved

Documents: SW-REQ-102

## Contract

Mongo aggregate persistence must retain the `Lists` projection generated by
`AnalyticsRecordAggregate.AsTimeUpdate`. Readback from both the per-org
aggregate collection and the shared mixed collection must populate fields such
as `Lists.APIKeys` with the identifier and counter values needed by listing
queries. The persisted model shape also preserves the restored list families
`APIID`, `OauthIDs`, `Geo`, `Tags`, `Errors`, `Endpoints`, `KeyEndpoint`,
`OauthEndpoint`, and `APIEndpoint`.

This is a child of SW-REQ-059. SW-REQ-059 owns per-org and mixed collection
routing; SW-REQ-102 pins the historical field-loss case fixed by commit
`d4d1cf7`.

## Evidence

- `analytics/aggregate_test.go:TestAnalyticsRecordAggregate_ListsBSONRoundTripPreservesRestoredFields`
  asserts the restored `Lists` families survive BSON marshal/unmarshal into
  `AnalyticsRecordAggregate`.
- `pumps/mongo_aggregate_test.go:TestDoAggregatedWritingWithIgnoredAggregations`
  writes aggregate data to both per-org and mixed collections and asserts
  readback contains `Lists.APIKeys` with the expected identifier and hit count.

## Known Issues

This requirement does not claim Mongo aggregate writes are atomic across the
two upsert phases or that backend timeouts are bounded. Those behaviors remain
tracked by existing Mongo aggregate KnownIssues and SW-REQ-059/SW-REQ-060
deferrals.
